# Premise Challenge

Before proposing solutions, challenge the premises and get explicit agreement.

Questions to resolve:
1. Is this the right problem? Could a different framing yield a simpler or more impactful solution?
2. What happens if nothing changes? Is this a real pain point or hypothetical?
3. What existing code, tools, or workflows already partially solve this?

Present premises as statements:
```
PREMISES:
1. ... — agree/disagree?
2. ... — agree/disagree?
3. ... — agree/disagree?
```
If the user disagrees, revise and loop back.
