# Implementation Alternatives (Mandatory)

Always provide **2–3 distinct approaches** before selecting a review mode.

```
APPROACH A: <name>
  Summary: <1–2 sentences>
  Effort:  [S/M/L/XL]
  Risk:    [Low/Med/High]
  Pros:    - ...
  Cons:    - ...
  Reuses:  <existing code/patterns>

APPROACH B: <name>
  ...

APPROACH C: <optional>
```

Rules:
- One approach must be **minimal viable** (fewest files, smallest diff).
- One approach must be **ideal architecture** (best long-term trajectory).
- If only one approach exists, explain why the others are not viable.
- Ask the user to choose before moving to mode selection.
