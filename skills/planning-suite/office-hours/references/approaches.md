# Alternatives Template

Always produce at least **two** distinct approaches.

```
APPROACH A: <name>
  Summary: <1–2 sentences>
  Effort:  [S/M/L/XL]
  Risk:    [Low/Med/High]
  Pros:    - ...
  Cons:    - ...
  Reuses:  <existing code/patterns>

APPROACH B: <name>
  ...

APPROACH C: <optional>
```

Rules:
- One approach must be **minimal viable** (fewest files, smallest diff).
- One approach must be **ideal architecture** (best long-term trajectory).
- Ask the user to choose; include a one-line recommendation if appropriate.
